BaytesSMC.SMCDiagnostics — Type
struct SMCDiagnostics{P, J, T<:AbstractFloat, G} <: BaytesCore.AbstractDiagnostics

SMC Diagnostics container, including diagnostics of kernels used in jittering step.

Fields

  • base::BaytesCore.BaseDiagnostics{Vector{P}} where P

    Diagnostics used for all Baytes kernels

  • ℓincrement::Float64

    Weighted average of incremental log weights - can be used for marginal or log predictive likelihood computation.

  • ℓweights::Vector{Float64}

    Cumulative log weights. If resampled, updated with jittered parameter - as weights will be used for tempering.

  • ℓweightsₜ::Vector{Float64}

    Log weights at current iteration, used for resampling criterion. NOT updated with jittered theta parameter as ℓweightsₜ depends on t-1 and jittered theta parameter come in at next iteration.

  • ℓweightsₙ::Vector{Float64}

    Normalized Log weights. They might be adjusted from previous iterations, so will differ from ℓweights. Will be set to log(1/N) if resampling applied.

  • jitterdiagnostics::Vector

    Diagnostics of jitter steps. If not resampled this iteration (i.e., resampled == false), contains jitterdiagnostics from previous step.

  • jittersteps::Int64

    Number of jittering steps

  • ρ::Vector{T} where T<:AbstractFloat

    Correlation from rejuvented continuous parameter

  • ESS::Float64

    ESS and accepted steps for SMC kernel

  • resampled::Bool

    Boolean if step has been resampled.

  • generated::Any

    Generated quantities specified for objective

BaytesCore.jitter! — Method
jitter!(_rng, particles, tune, data, temperature)

Jitter θ particles with given kernels. This is performed in 2 stages:

  1. kernel is updated with new data and shuffled particles, then one proposal step is performed.
  2. If more than one jitterstep has to be performed, previous results might be captured depending on the kernel.

BaytesSMC.SMCweight — Method
SMCweight(_rng, objective, algorithm, cumweightsₜ₋₁)

Compute cumulative and incremental weight of objective at time/iteration t, given weight at t-1. Incremental weight will be used as particle weight for resampling. Cumulative weight will be used to adapt temperature.

If temperature is constant, this function can be overloaded with your ModelName if incremental weight can be computed independent of previous weight, which speeds up computation. cumweightsₜ is not needed in this case.
